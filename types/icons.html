icon_library.sym

Icons are tiny signage--simple, bold, and readable like symbols on old equipment. They should feel hand-drafted but precise, like they were plotted on a grid.

icon_style.sty

This set is a single pixel-sprite sheet (media.png) with 32×32 cells. Icons should feel like hardware glyphs: chunky, high-contrast, and snapped to a strict grid.

rule guideline
grid design on a 32×32 pixel cell; align edges to whole pixels (no subpixel curves)
stroke prefer 1–2px strokes; keep weight consistent across the whole set
silhouette readable at a glance; remove detail until the shape survives at 1× size
palette limit colours per icon (2–4); bias toward system inks (--color-primary, --color-accent) + neutrals
padding keep ~2–4px breathing room from the cell edge so icons don’t “touch the frame”
motion if animating, do it by swapping cells (sprite frames), not by blurring/transforming the bitmap

icon_sizing_and_usage.size

Because the set is pixel-based, scale in whole-number steps. Keep the icon cell 32px, then scale to 64px or 96px with CSS. For tap targets, add padding around the icon (don’t scale the bitmap to “fake” a hit area).

size how use notes
32px .icon inline glyphs, lists, UI chrome default; stays crisp at 1×
64px transform: scale(2) feature buttons, empty states, demo panels integer scaling keeps pixels sharp
96px transform: scale(3) hero toggles / “hardware” controls use sparingly; add breathing room
tap target padding around icon click/tap controls aim for ~44px minimum interactive area

icon_usage_examples.ecwq

Use icons with labels for clarity in primary actions, and icon-only buttons only when the meaning is universal. In dense UIs, treat icons like shorthand, not decoration.

icon_accessibility.acc

Provide text alternatives (visible labels or aria-label), keep tap targets generous, and maintain contrast. If it's not readable on a dim CRT, it's not readable.

css_implementation.css

Icons are pulled from a sprite sheet by setting --x and --y per glyph. Create semantic classes that map to coordinates.

/* our sprite base */
.icon
{
    width: 32px;
    height: 32px;
    background-image: url('media.png');
    background-size: 128px 160px; /* 4 cols x 5 rows */
    background-position: calc(var(--x) * -32px) calc(var(--y) * -32px);
    image-rendering: pixelated;
}

/* semantic mapping */
.ic-search { --x: 2; --y: 2; }
.ic-settings { --x: 2; --y: 3; }
