layer_stack.str

Layers are the depth rules of the interface: what sits behind, what sits on top, and what must always win focus. In CSS this is mostly controlled by position + z-index (and the stacking contexts they create).

layer_history.hist

Think of it like a light table: base paper (background), the drawing (content), acetate notes (elevated UI), then a full-sheet overlay (modal/menu) when you need attention to snap to one place.

layer role what it contains why it exists
1 background page canvas, base surfaces sets atmosphere without competing with content
2 content reading regions, main layout blocks keeps information stable and scannable
3 elevated_ui cards, controls, callouts adds emphasis for interactive objects
4 overlay dialogs, menus, toasts, focus states forces priority and reduces distraction

applied_layering_example.demo

The demo below shows the four roles stacked together. Notice that overlay wins, elevated UI sits above content, and content sits above the base canvas.

layer_tokens.tok

Tokens make layering predictable: engineers can place things without guessing and designers can review stacks without hunting through random z-index numbers.

token value use
--layer-base 0 background canvas and foundational surfaces
--layer-content 10 primary content regions and reading surfaces
--layer-elevated 100 cards, controls, and emphasized interface objects
--layer-overlay 1000 dialogs, popovers, menus, and temporary overlays

css_implementation.css

Keep layering predictable by using z-index tokens. Avoid random values like 9999; reserve overlays for true focus states.

:root
{
    --layer-base: 0;
    --layer-content: 10;
    --layer-elevated: 100;
    --layer-overlay: 1000;
}

.card
{
    position: relative;
    z-index: var(--layer-elevated);
}

.modalBackdrop
{
    position: fixed;
    inset: 0;
    z-index: var(--layer-overlay);
}
