Typography is the art and technique of arranging type to make written language legible, readable, and appealing when displayed. It involves selecting typefaces, point sizes, line lengths, letter-spacing, and adjusting the space between groups of letters.

font_pairing.pair

Typography should read like a labeled cassette deck: clear, confident, and a little nostalgic. The goal is legibility first, vibe second--but we still keep the analog charm.
This design system will use a two-font pairing: VHS for headlines and labels, and Typewriter Mono for body text and UI details. The fonts should feel like they belong together, like two pieces of hardware from the same era. The loudness of VHS screams the title, while IBMPlexMono keeps things legible at a lower level.

role font weight / style used for
display / titles Vhs (VHS.ttf) display (single cut) page titles, section headers, module labels
body / interface IBM Plex Mono (IBMPlexMono) regular / semibold / bold paragraphs, tables, navigation, metadata, UI copy

type_scale.ecc

Use a stepped scale like a typographer's ruler: a few sizes that repeat often. Consistent steps keep hierarchy obvious, like track numbers on a mixtape.

role selector font size line-height usage
display h1 Vhs 60px normal primary page title
headline h2 Vhs 40px normal section headers
category h4 Vhs 20px normal module labels, overlines
body p IBM Plex Mono 18px 1.5 reading text, descriptions
label .checklist p IBM Plex Mono 14px normal navigation headers, metadata
meta span IBM Plex Mono 75% inherit suffixes, extensions, muted info

typography_principles.ewfs

Prioritize contrast, rhythm, and line length. Keep paragraphs short, avoid long all-caps runs, and let whitespace do the talking like clean margins on a printed flyer.

principle rule
font pairing use only Vhs for titles and IBM Plex Mono for reading/interface text
hierarchy prefer size + spacing changes over adding new font families or styles
readability keep body line-height comfortable (1.5+) and cap line length (780px max already)
restraint avoid long all-caps runs in body; use semibold/bold sparingly for emphasis
effects safety if CRT effects reduce legibility, toggle them off for reading-heavy pages

css_implementation.css

Typography is token-driven: one display font, one mono body font, and a small role-based scale.

:root
{
    --font-display: 'Vhs', sans-serif;
    --font-mono: IBMRegular, monospace;

    --text-display: 60px;
    --text-headline: 40px;
    --text-category: 20px;
    --text-body: 18px;
    --lh-body: 1.5;
}

h1, h2, h4 { font-family: var(--font-display); }
p { font-family: var(--font-mono); font-size: var(--text-body); line-height: var(--lh-body); }
